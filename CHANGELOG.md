# 更新日志

本项目遵循语义化版本。日期使用 `YYYY-MM-DD` 格式。

## v0.4.1 - 2026-07-22

### 修复

- 修复 psycopg 3 批量写入错误，改为通过 Cursor 调用 `executemany()`。
- 增加 PostgreSQL 批量写入调用方式的回归测试。

## v0.4.0 - 2026-07-22

### 新增

- 将插件运行时存储切换为 PostgreSQL。
- 增加 PostgreSQL 连接池、自动建表、字段约束和 Schema 版本记录。
- 使用 JSONB 保存知识关键词与来源，并增加 GIN 索引。
- 增加 `/存储状态` 管理指令。
- 增加 PostgreSQL Docker Compose 示例。
- 增加 SQLite 历史数据迁移工具。

### 调整

- 保留知识主题引用的原始证据消息，不参与普通消息过期清理。
- SQLite 仅作为旧数据迁移来源和测试仓储。

## v0.3.0 - 2026-07-22

### 新增

- 增加 `/分类消息`，使用小模型逐条执行 `keep/discard` 分类。
- 增加知识主题、消息分类和主题消息关联存储。
- 增加 `/知识主题` 和 `/总结主题` 指令。
- 根据有效消息数量自动维护 `collecting`、`ready` 和 `summarized` 状态。
- 支持分类模型与总结模型分别配置。

## v0.2.0 - 2026-07-22

### 新增

- 增加结构化 JSON 知识抽取和严格字段校验。
- 增加来源消息验证、证据回填、可信度计算和 SHA-256 去重。
- 增加候选知识、抽取任务和审核状态存储。
- 增加 `/候选知识`、`/通过知识` 和 `/拒绝知识` 指令。

## v0.1.0 - 2026-07-22

### 新增

- 创建 AstrBot 官方模板插件结构。
- 保存群聊纯文本消息并支持灵活时段查询。
- 增加按需群聊总结和 QQ 纯文本输出。
- 增加 NapCat 群历史消息补录与消息 ID 去重。
